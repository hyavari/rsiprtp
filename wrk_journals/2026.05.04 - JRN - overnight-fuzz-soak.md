## Plan

This target is the new round-trip oracle that surfaced the

### Why round-trip, not differential

- `sip_message_parse_diff` already had an 8h M11 campaign and 17 pinned
  divergences. Marginal value of further compute is low.
- `sip_message_roundtrip` is brand new. First real fuzz run found a
  bug at ~5 min / 6.8M execs. Probability of more is non-trivial.

### Supervisor protocol (per memory: sub-agents drop their wait on
long processes)

- **Supervisor** (me) owns the long-running fuzz wait. Bash
  `run_in_background` + Monitor for the event stream. Do NOT delegate
  the wait itself to a sub-agent.
- **Sub-agents** are dispatched per crash to:
  1. Investigate (Explore agent, scope: read the crash-log file,
     report shape).
  2. Diagnose + propose fix (general-purpose agent, scope: read the
     parser/serializer and propose a minimal fix).
  3. Implement the fix (general-purpose agent, scope: edit the file,
     add unit test, add diff-pin if needed).
  4. Review (code-reviewer skill or Plan agent — TBD per crash).
  5. Markup the journal (this file).
- Each crash gets its own short subsection in this journal.

### Wrapper behavior

`fuzz_wrapper.sh` lives at the repo root (gitignored — see below).
It does:

1. Read deadline = launch_time + 8h.
2. Loop:
   - Compute remaining seconds; exit if ≤ 60.
   - Run `cargo +nightly fuzz run sip_message_roundtrip -- -max_total_time=R -timeout=10 -rss_limit_mb=512`
     with stderr+stdout teed to `wrk_journals/fuzz_logs/run_NNN.log`.
   - On non-zero exit: copy log + any artifact to
     `wrk_journals/fuzz_triage/crash_NNN/` and emit
     `CRASH run=NNN exit=X queue=crash_NNN` to wrapper stdout (the
     Monitor event).
   - On clean `-max_total_time` exit: emit
     `CLEAN run=NNN runs=R` and continue (next loop iter computes new
     deadline budget).
3. Final stdout line: `DONE total_runs=N total_crashes=M`.

The MSVC `bin\Hostx64\x64` directory (with
`clang_rt.asan_dynamic-x86_64.dll`) is prepended to PATH at script
start. Path:
`C:\Program Files\Microsoft Visual Studio\18\Enterprise\VC\Tools\MSVC\14.50.35717\bin\Hostx64\x64`
