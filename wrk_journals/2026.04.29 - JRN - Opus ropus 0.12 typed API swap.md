**ropus contract discovery (worth recording for the bridge team):**
`set_bitrate(Bitrate::Bits(N))` with `N > 750_000 * channels` does **not** return `Err` — ropus silently clamps to `750_000 * channels` and returns `Ok`, mirroring the libopus C reference. Only `Bits(N)` with `N <= 0` (excluding `Auto`/`Max` sentinels) returns `BadArg`. Source: `ropus-0.12.0/src/opus/encoder.rs:3466-3484`. Implication for the bridge: a clamp-then-call pattern in the controller is *not* required for safety, but is desirable for telemetry — if you want to see "I asked for 10Mbps and got 750k", read back `bitrate()` after setting. The new test `test_set_bitrate_huge_clamped` asserts this contract so any future ropus change to "reject" instead of "clamp" surfaces immediately.
