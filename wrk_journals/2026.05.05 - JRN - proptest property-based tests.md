# JRN — Property-based tests via proptest

**Date opened:** 2026-05-05 (22:55 UTC)
**Supervisor:** Claude (Opus 4.7), acting as team lead
**Source:** Suggestion from another agent — "Generators producing structurally-valid SIP messages catch corner combinations libfuzzer's bit-mutation will never reach in reasonable time, and shrinking on failure is invaluable for triage."

## Goal

Add `proptest`-based property tests that complement the existing libfuzzer corpus. Where libfuzzer mutates bits, proptest generates structurally-valid inputs and shrinks on failure — strictly additive value, not a replacement for fuzzing.

## Workstream split

Three parallel tracks, mirroring existing fuzz-target boundaries so reviewers can compare like-for-like:

| Track | Scope | Reuses |
|------|------|--------|
| **A — SIP message** | `sip::parser::Message` round-trip property | `parser_roundtrip_oracle::assert_roundtrip_fixed_point` |
| **B — SIP typed headers** | `Via`, `From`, `To`, `Contact`, `CSeq`, `SipUri` parse↔serialize | per-typed-header `to_string` (where present) |
| **C — SDP session** | `SessionDescription` parse↔serialize | `sdp_roundtrip_oracle::assert_roundtrip_fixed_point` |

## Process

For each track:
1. **HLD agent (Plan)** — drafts an HLD under `wrk_docs/2026.05.05 - HLD - proptest <track>.md`. Must define generator strategies (with explicit ranges/biases), oracles invoked, shrinking expectations, file layout, and dev-dep additions.
2. **Supervisor review** — iterate with the HLD agent until I sign off. Reject vague/under-specified designs.
3. **Implementation agent** — writes the proptest module per the signed HLD.
4. **/code-reviewer agent** — invokes the `code-reviewer` skill on the diff.
5. **/code-review-apply-feedback agent** — applies must-fix items.
6. Commit on green.

Per the supervisor sub-agent wait anti-pattern (memory file `feedback_supervised_subagent_waits.md`), **the supervisor drives long-running waits via `Bash run_in_background` / Monitor**, not the sub-agent.

**Track B** (`tests/proptest_sip_typed.rs`)
- Landed 619 LOC; 13 properties, all pass; runtime 0.26s.
- Implementation agent's quota exhausted before final report-back — but the file had landed, was clippy-clean, and ran end-to-end. Supervisor took it from there.
- **Real finding** (kept in code as the documented exclusion at `display_name_quoted_inner`):

  Failing input shrunk to `s = "\"*\" <sip:a>"`. Trace:
  1. First parse succeeds → `Contact::Addr` with display_name = `*`.
  2. `Contact::Display` re-emits without quotes → `* <sip:a>`.
  3. Reparse sees the leading `*`, routes to the wildcard-Contact branch, fails with `InvalidHeader("trailing data after wildcard Contact")`.

  Root cause: `Contact::Display` (in `src/sip/parser/typed/contact.rs`) does not re-quote display names containing characters that the Contact-form disambiguator (`*`) interprets specially. Either Display must re-quote on emit, or the parser must require *bare* `*` (no surrounding quotes/whitespace) for the wildcard form.

  Per HLD anti-scope, the proptest landing did not fix the production bug. Workaround: `*` added to the `display_name_quoted_inner` filter exclusion. Inline docstring documents the bug, the trigger, and points back here.

  **Follow-up TODO:** triage the `Contact::Display` ambiguity and decide whether to fix in `Display` or `parse`. Removing `*` from the exclusion filter once the production code is fixed will re-enable that generator branch.

**Track C** (`tests/proptest_sdp_session.rs`)
- Landed 379 LOC; 2 properties, both pass; runtime 0.12s.
- HLD deviations (all defensible per Stage-4 review):
  - `MediaBuilder::audio` is the only public constructor; agent built audio then mutated `media_type` post-build (parser/serializer treat media_type independently of attrs/protocol/formats — no invariant violated).
  - `bandwidth`, media-level `c=`, ICE attrs set by direct field assignment (builder doesn't expose these — the path-(b) escape hatch the HLD documents).
  - `session_id`/`session_version` passed as `u64` directly (no string round-trip — agent recognized `u64::MAX.to_string().parse::<u64>()` is lossless and dropped the indirection).
  - `arb_other_attr` excludes `Direction` (builder always emits exactly one — generating a second would create a duplicate that clutters the AST).
- Real findings: none.
