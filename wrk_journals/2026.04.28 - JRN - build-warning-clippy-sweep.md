## Clippy findings & fixes
All findings were genuine quality issues. None indicated missing code or bad
merges; most were either:
- new clippy lints in 1.94 (`io_other_error`, `unnecessary_map_or`,
  `useless_vec` for `Vec` only used as slices), or
- intentional 100%-coverage tests that exercise auto-derived `Clone`/`Copy`
  (clippy correctly notes the call is redundant — kept the test, added a
  scoped `#[allow(clippy::clone_on_copy)]` with a comment), or
- pre-existing test scaffolding that had real defects (see "Defects" below).

### Defects (not just lint noise)
1. `crates/mdsiprtp/tests/asterisk_integration.rs:487` —
   `assert!(received_100 || received_18x || true, ...)`. The trailing `|| true`
   short-circuited the assertion to always pass. Inspecting the loop showed
   the only normal exit is the 200/ACK/BYE/200-OK happy path, so the assert
   itself was redundant once we accept that loop exit means success. Removed
   the broken assert and the now-unused tracking flags.
2. `crates/mdsiprtp/examples/voicemail.rs:142` — `match call.state { Recording
   => {...} _ => {} }` flagged as "match used for equality". Converted to an
   `if call.state == VoicemailState::Recording` (state already derives `Eq`).
3. `crates/mdsiprtp/tests/stack_instance/mod.rs:405` — `for (call_id, _) in
   &self.outbound_calls { ...; break; }` iterating a map's keys with an
   unconditional break. Replaced with
   `if let Some(call_id) = self.outbound_calls.keys().next() { ... }`.
4. `crates/mdsiprtp-ice/src/agent.rs:3068, 3260` — two `for _ in 0..2 { ...;
   break; }` loops in mock STUN servers. The body always handles exactly one
   request and then breaks — the loop is dead. Removed the loop wrapper.
5. `crates/mdsiprtp-media/src/mixer.rs:992` —
   `assert!(s >= i16::MIN); assert!(s <= i16::MAX);` on `i16` samples are
   tautological and don't actually verify the AGC clipping. Reworked the test
   to drive the AGC into clamp territory (target_level=1.5, max_gain=10.0)
   and assert the saturated samples equal `i16::MAX`/`i16::MIN`.
6. Several files: `Vec`s constructed only to take a slice (`&vec![...]`) or
   only mutated as a slice (`let mut targets = vec![...]; targets.sort_by(...)`).
   Switched to arrays where the type permits (arrays auto-deref to slices and
   support `sort_by`).
7. Several files: manual `if s.starts_with(prefix) { s[prefix.len()..]... }`.
   Switched to `s.strip_prefix(prefix)`.
8. `crates/mdsiprtp/tests/stack_instance/mod.rs:559` — `.split('@').last()` on
   a `DoubleEndedIterator` is `O(n)` for what should be `O(1)`. Switched to
   `.next_back()`.
9. `crates/mdsiprtp-sip/src/message.rs:2395` — duplicate assert
   (`!is_empty()` followed by `len() > 0`). Removed the redundant second.
10. `crates/mdsiprtp-session/src/{hold,transfer}.rs` — `Default::default()`
    followed by field assignment. Switched to struct-update syntax.
11. Many `std::io::Error::new(ErrorKind::Other, msg)` → `std::io::Error::other(msg)`
    across `mdsiprtp-srtp`, `mdsiprtp-ice`, `mdsiprtp-media`, `mdsiprtp-transport`,
    `mdsiprtp-rtp`. New 1.94 lint.
12. `mdsiprtp-srtp/src/context.rs:432` — `for i in 12..172 { rtp[i] = ... }`
    where the body just indexes by `i`. Switched to
    `for (i, byte) in rtp[12..172].iter_mut().enumerate()`.
13. `crates/mdsiprtp-ice/src/stun.rs:1527-1528` — XOR-mapped-address test had
    documentary `0 ^ 0xXX` literals that fired `identity_op`. Kept the formula
    intent in a comment, used the literal magic-cookie bytes directly.
14. `crates/mdsiprtp-dialog/src/state.rs:977,993` —
    `DialogInfo::detect_secure_transport(&invite)` where `invite` was already
    `&SipRequest`. Removed the extra `&`.

### Coverage tests retained with allow attrs
Tests of the form `let cloned = x.clone();` on `Copy` types are explicit
coverage of the auto-derived `Clone` impl. Each affected test got
`#[allow(clippy::clone_on_copy)]` with a `// exercise derived Clone for
coverage` comment so the intent is documented:
- `mdsiprtp-srtp/src/dtls.rs` (DtlsRole, FingerprintHash, SrtpProfile, DtlsState)
- `mdsiprtp-rtp/src/dtmf.rs` (DtmfDigit)
- `mdsiprtp-transaction/src/client/invite.rs` (State)
- `mdsiprtp-ice/src/candidate.rs` (CandidateType, Transport)
- `mdsiprtp-dialog/src/invite.rs` (Role)
- `mdsiprtp-dialog/src/manager.rs` (DialogHandle)
- `mdsiprtp-transport/src/traits.rs` (TransportProtocol)
- `mdsiprtp-transport/src/resolver.rs:2391` — replaced `transport.clone()`
  inside a struct literal with `transport` (Copy lets us just consume).
