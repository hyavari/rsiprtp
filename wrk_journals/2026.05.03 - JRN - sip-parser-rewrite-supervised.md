# Goal

Implement the 12-milestone parser rewrite from the HLD using an agent team.
Supervisor (this file's author) does **no implementation, testing, or
debugging directly** — purely orchestration, review-of-review, and
direction. Aim: 100% feature complete, fully diff-tested against rsip,
ready for a 0.4.0 release.

# Agent team

Each milestone runs through this pipeline:

1. **Implementation agent** (general-purpose) — writes code per the
   milestone scope. TDD where appropriate (tests first, then impl). Runs
   touched-path tests to verify before returning.
2. **Code reviewer** (using `code-reviewer` skill) — reviews the diff for
   quality, correctness, conventions.
3. **Apply-feedback agent** (using `code-review-apply-feedback` skill) —
   acts on the review's REQUIRED items and any SUGGESTED ones in scope.
4. **Devils advocate** (general-purpose) — challenges decisions made by
   the impl + review + apply-feedback chain. Reviews the supervisor's
   stage-gate decision too. Returns either "endorse" or specific concerns
   to address before commit.
5. **Supervisor** (me) — verifies, commits the milestone atomically,
   updates this journal, advances.

# Anti-patterns to avoid (from prior overnight pipelines)

- **Sub-agent zombie wait.** Sub-agents that kick off long processes
  (full_test, fuzz) and exit expecting system notification return
  prematurely. Either wrap waits in supervisor-side `Bash run_in_background`
  / `Monitor`, or instruct the sub-agent that the wait IS its job. Prefer
  the former.
- **`[N/7] PASS` grammar.** full_test driver doesn't print explicit
  `[7/7] PASS`. Detect completion via the dated HTML report file landing
  in `crates/rsiprtp/tests/results/` or via the `cargo run` exit code.
- **Stale git-status snapshots.** Multi-agent activity may have advanced
  HEAD between the system context's snapshot and the supervisor's first
  tool call. Refresh `git log` first thing.
