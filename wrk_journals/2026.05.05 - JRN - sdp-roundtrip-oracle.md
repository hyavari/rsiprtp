### Stage 5 ‚Äî devils-advocate review + apply (about to commit)

Devil's-advocate review challenged the work hard. Three real findings were taken; one was triaged out (UTF-8 fuzz short-circuit ‚Äî cosmetic, devil's-advocate himself called it not load-bearing) and one was a false positive (`cargo fuzz build` was actually verified in stages 3 and 4).

Findings applied:

1. **SIP canonical-input sanity test** ‚Äî the SDP equivalent was fixed in Stage 2; the SIP version was deliberately left as out-of-scope. Devils-advocate argued (correctly) that "mirror SIP" was the whole HLD's premise, so leaving SIP broken is a known landmine. Apply-feedback agent reformatted the byte literal and added a self-verifying `assert_eq!(m1.to_bytes(), canonical)` assertion. Honest correction discovered: Rust's `\<newline>` line-continuation strips leading whitespace *before* the parser sees it, so the original "indentation breaks canonicity" framing was wrong ‚Äî but the self-verifying assertion still has value as a regression guard.

2. **Coverage gaps in fixture corpus** ‚Äî HLD enumerates ~14 lossy normalizations; only 3-4 had fixtures. Added 6 new fixtures: `media_type_other.sdp`, `num_ports.sdp`, `lossy_timing.sdp`, `multi_c_per_media.sdp`, `misordered_after_m.sdp`, `session_b_dropped.sdp`. Each with matching `rt_sdp_*` test.

3. **Bandwidth-sort regression test was missing** ‚Äî original `bandwidth.sdp` had only 2 keys (AS, CT) which couldn't expose the HashMap-seed flake. Added `bandwidth_collision.sdp` with 5 keys per media (TIAS/AS/CT/RR/RS) and `rt_sdp_bandwidth_sort_is_deterministic` test running the oracle 20√ó over fresh instances. **Confirmed load-bearing** by temporary-revert: without sort, fixture fails on first iteration (e.g., audio block emits `AS,RS,RR,TIAS,CT` on t2 but `RS,RR,AS,TIAS,CT` on t3). Sort restored; test passes.

Final test counts: SDP roundtrip 16/16 (9 original + 7 new), SIP roundtrip 23/23, clippy clean.
