# Production-Quality Test Coverage Expansion

## Objective
Expand test coverage to production-level quality for the mdsiprtp SIP/RTP stack. Focus on RFC compliance, security edge cases, fault tolerance, and comprehensive unit testing of all critical paths.

## Current State
- 1548 total tests across workspace
- Integration tests: 88 tests in baresip_integration
- Good coverage of basic functionality
- Gaps in RFC compliance, security testing, fault injection, and edge cases

## Priority Areas

### 1. RFC Compliance Testing (HIGH PRIORITY)

#### SIP RFC 3261 Compliance
Location: `crates/mdsiprtp-sip/src/` and new test module

**Tests to add:**
- Header parsing edge cases (folding, UTF-8, quoted strings, comments)
- URI parsing (all schemes: sip, sips, tel; user-info encoding; parameters)
- Via header branch parameter validation (magic cookie z9hG4bK)
- CSeq method matching validation
- Content-Length header handling (missing, zero, mismatch)
- Request-URI vs To header consistency
- Record-Route and Route header processing
- Max-Forwards decrement and zero handling
- Require/Supported/Unsupported header negotiation
- 100rel (PRACK) handling if supported
- Timer C (INVITE transaction timeout) compliance

#### SDP RFC 4566/RFC 3264 Compliance
Location: `crates/mdsiprtp-sdp/src/`

**Tests to add:**
- Offer/Answer model state machine tests
- Media line ordering and port allocation
- Codec preference ordering (m= line)
- Attribute inheritance (session vs media level)
- Bandwidth modifier handling (AS, CT, TIAS)
- ICE attribute parsing (ice-ufrag, ice-pwd, candidate)
- RTCP attribute (a=rtcp:) parsing
- Direction attributes (sendonly, recvonly, sendrecv, inactive)
- Connection address handling (IPv4/IPv6, multicast)
- Origin session-id and version increment rules

#### RTP RFC 3550 Compliance
Location: `crates/mdsiprtp-rtp/src/`

**Tests to add:**
- Sequence number wraparound handling (65535 -> 0)
- Timestamp wraparound handling (32-bit)
- SSRC collision detection and resolution
- CSRC list handling (contributing sources)
- Padding bit handling
- Extension header parsing
- RTCP compound packet requirements
- RTCP SR/RR report generation timing
- RTCP BYE packet handling
- RTP/RTCP port pairing (even/odd)
