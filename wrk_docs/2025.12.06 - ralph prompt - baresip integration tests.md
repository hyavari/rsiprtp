## Implementation Tasks

### Task 1: Baresip Process Controller (`framework/baresip.rs`)

Create a Rust struct that manages baresip as a subprocess:

```rust
pub struct BaresipInstance {
    process: Child,
    ctrl_socket: PathBuf,  // baresip control socket
    sip_port: u16,
    rtp_port_range: (u16, u16),
    config_dir: TempDir,
}

impl BaresipInstance {
    /// Spawn baresip with test configuration
    pub async fn spawn(config: BaresipConfig) -> Result<Self>;

    /// Send command via control socket (e.g., "/dial sip:...", "/hangup")
    pub async fn command(&self, cmd: &str) -> Result<String>;

    /// Wait for specific event in baresip output
    pub async fn wait_for_event(&self, pattern: &str, timeout: Duration) -> Result<()>;

    /// Get current call state
    pub async fn call_state(&self) -> Result<CallState>;

    /// Graceful shutdown
    pub async fn shutdown(self) -> Result<()>;
}
```

Key baresip commands to support:
- `/dial sip:user@host` - Initiate outgoing call
- `/hangup` - End current call
- `/hold` - Put call on hold
- `/resume` - Resume held call
- `/transfer sip:target` - Blind transfer
- `/accept` - Accept incoming call
- `/reject` - Reject incoming call

#### 4.2 Basic Call Tests (`scenarios/basic_call.rs`)

```rust
#[tokio::test]
async fn test_outgoing_call_to_baresip() {
    let config = TestConfig::with_available_ports();
    let mut endpoint = TestEndpoint::new(config.clone()).await?;
    let baresip = BaresipInstance::spawn(config.baresip_config()).await?;

    // mdsiprtp calls baresip
    let call = endpoint.call(&format!("sip:test@127.0.0.1:{}", config.baresip_sip_port)).await?;

    // baresip accepts
    baresip.command("/accept").await?;
    baresip.wait_for_event("call established", Duration::from_secs(5)).await?;

    // Verify RTP is flowing
    tokio::time::sleep(Duration::from_secs(2)).await;
    let stats = endpoint.rtp_stats(&call);
    assert!(stats.packets_received > 0, "Should receive RTP packets");

    // Hangup
    endpoint.hangup(&call).await?;
    baresip.wait_for_event("call closed", Duration::from_secs(5)).await?;
}

#[tokio::test]
async fn test_incoming_call_from_baresip() {
    let config = TestConfig::with_available_ports();
    let mut endpoint = TestEndpoint::new(config.clone()).await?;
    let baresip = BaresipInstance::spawn(config.baresip_config()).await?;

    // baresip calls mdsiprtp
    baresip.command(&format!("/dial sip:test@127.0.0.1:{}", config.local_sip_port)).await?;

    // mdsiprtp accepts
    let incoming = endpoint.wait_for_incoming(Duration::from_secs(5)).await?;
    let call = endpoint.accept_call().await?;

    // Verify call established
    baresip.wait_for_event("call established", Duration::from_secs(5)).await?;

    // baresip hangs up
    baresip.command("/hangup").await?;

    // Verify BYE received
    endpoint.wait_for_hangup(&call, Duration::from_secs(5)).await?;
}
```

```rust
pub fn assert_rtp_flowing(stats: &RtpStats) {
```

5. **Fifth**: Add more scenarios incrementally

## Key Considerations

1. **Port Management**: Use dynamic port allocation to avoid conflicts
2. **Process Cleanup**: Ensure baresip is killed even on test failure (use Drop)
3. **Timeouts**: All async operations need reasonable timeouts
4. **Logging**: Capture baresip stdout/stderr for debugging
5. **Isolation**: Each test should be independent, spin up fresh instances
6. **Skip Conditions**: Skip tests if baresip not installed (`#[ignore]` with check)

## Success Criteria

- [ ] Can spawn/control baresip from Rust tests
- [ ] Basic call (INVITE → 200 OK → ACK → BYE) works bidirectionally
- [ ] RTP media flows and is validated
- [ ] DTMF works in both directions

- [ ] All tests pass in CI
