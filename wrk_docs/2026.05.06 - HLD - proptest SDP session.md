## 1. Scope

Drive the existing SDP round-trip fixed-point oracle from a proptest strategy that generates **structurally-valid SDP session descriptions**, where "structurally valid" means: parses successfully on the first call to `SessionDescription::parse`. The oracle handles the lossy-first-pass normalizations defined in the SDP oracle HLD; we just need the input to make it past parse #1.

This complements `fuzz/fuzz_targets/sdp_session_roundtrip.rs` (libfuzzer bit-mutation, mostly malformed inputs that the oracle silently discards) by exercising the *valid* corner of the input space — the half of the oracle that bit-mutation rarely reaches.

### What we are testing
- `sdp_roundtrip_oracle::assert_roundtrip_fixed_point` against generated bytes.
- The full pipeline `SdpBuilder::build → Display::to_string → bytes` plus, for fields the builder does not expose (port range, multiple bandwidth keys, session-level attributes, ICE attrs), direct mutation of the built `SessionDescription` before `to_string`.

### Non-goals (anti-scope)
- **No SDP offer/answer negotiation.** `create_answer`, `process_answer`, `Codec`, `NegotiatedMedia` (`crates/rsiprtp/src/sdp/negotiation.rs`) are not in scope. Negotiation has its own `==` semantics (selected codec list ordering, rejected-stream ports, direction inversion) that deserve their own property-based oracle and HLD.
- **No exhaustive ICE/SRTP attribute combinatorics.** A representative subset of ICE attrs (`ice-ufrag`, `ice-pwd`, `candidate`, `rtcp-mux`) only — full RFC 8839 candidate-line space and DTLS-SRTP fingerprint variants are excluded.
- **No non-RTP `m=` lines.** Generators emit `RTP/AVP` and `RTP/SAVP` only; protocols like `UDP/TLS/RTP/SAVPF`, `TCP/MSRP`, `DCCP/RTP/AVPF` are out — they need their own typed payload-type/format space which we do not have generators for.
- **No fixing of latent serializer/normalizer bugs.** If the property exposes one (e.g., `num_ports` drop, session-level `b=` drop, `MediaType::Other` collapse, CRLF), we file it; we do not fix it under this HLD.

## 2. Construction path

**We commit to path (a): `SdpBuilder` → `to_string()` → bytes → oracle.**

Rationale:
- Path (a) tests the public-API path. Anything an external user can produce by composing builder methods must round-trip; that is the contract we ship.
- Path (b) (constructing `SessionDescription` directly) bypasses builder invariants like the always-emitted direction attribute (`builder.rs:252-260`), the `addr_type` derivation from local IP (`:81-85`), and the always-`IN`/`IP4|IP6` net-type. Bypassing those would generate ASTs that the builder itself cannot produce, biasing failures away from the builder contract.
- Path (b) is *partially* used as an escape hatch: the builder does not expose `bandwidth`, `num_ports`, session-level attributes, media-level connections, or ICE attrs. For those, we mutate the built `SessionDescription` between `.build()` and `.to_string()`. This is documented in §3.7.

## 3. Strategy tree

We use proptest's `prop_compose!` and `Strategy::prop_map` rather than `derive(Arbitrary)`. The `proptest-derive` macro would force us to add it as an extra dev-dep and would not give us the constraints we want (e.g., "never zero-length formats", "valid IP addresses only"). All strategies live in the test file as helper functions.

Generation throughout uses `prop_oneof![Just(...), ...]` for finite enums and small bounded ranges — proptest shrinks toward the *first* arm of `prop_oneof!`, so the first arm is always the "simplest" value (empty vec, smallest int, default enum variant).

### 3.1 Origin (`o=`)

```rust
fn arb_origin_username() -> impl Strategy<Value = String> {
    // RFC 4566 token: 1*(safe-chars). We use a small ASCII subset
    // that the parser's split_whitespace can never split. NEVER empty.
    "[A-Za-z0-9_.\\-]{1,12}"
}

fn arb_numeric_string(max: u64) -> impl Strategy<Value = String> {
    // session-id and session-version are stored as String but
    // serialize lossless if they round-trip as digits. Generating
    // (0..=max).prop_map(|n| n.to_string()) keeps us in the lossless
    // numeric subset.
    (0u64..=max).prop_map(|n| n.to_string())
}

fn arb_ip4() -> impl Strategy<Value = std::net::Ipv4Addr> {
    // Small valid set — proptest shrinks toward 0.0.0.0. We do NOT
    // generate the full u32 space; nothing about the round-trip
    // depends on which exact octets we pick, and small addresses
    // shrink to a more readable failure.
    prop_oneof![
        Just(Ipv4Addr::new(0, 0, 0, 0)),
        Just(Ipv4Addr::LOCALHOST),
        Just(Ipv4Addr::new(192, 168, 1, 1)),
        Just(Ipv4Addr::new(10, 0, 0, 1)),
        Just(Ipv4Addr::new(203, 0, 113, 1)),
    ]
}

fn arb_ip6() -> impl Strategy<Value = std::net::Ipv6Addr> {
    prop_oneof![
        Just(Ipv6Addr::LOCALHOST),                       // ::1
        Just(Ipv6Addr::UNSPECIFIED),                     // ::
        Just(Ipv6Addr::new(0x2001, 0xdb8, 0, 0, 0, 0, 0, 1)),
    ]
}

fn arb_ip() -> impl Strategy<Value = IpAddr> {
    prop_oneof![arb_ip4().prop_map(IpAddr::V4), arb_ip6().prop_map(IpAddr::V6)]
}
```

Net-type and addr-type are not generated independently: the builder derives them from `local_addr` (`builder.rs:81-85`), so `IN` and `IP4`/`IP6` come for free.

### 3.2 Session name (`s=`)

The oracle allows defaulted-`s=` to renormalize on the first parse (HLD's "Defaulted session name" entry). Two viable choices:

- **(A)** Always emit a real, non-empty name from a small token set.
- **(B)** Sometimes emit `None` (i.e., omit the `s=` line) and rely on the oracle to absorb the `s=` → `s=-` collapse on the first round-trip.

**Decision: (A).** The proptest path tests the builder, and the builder always sets a session name (defaulting to `"-"`); we cannot ask the builder to omit `s=`. The "missing `s=`" normalization is already covered by the static-fixture driver (`rt_sdp_missing_session_name`). Generating it here would require dropping path (a) for that case, which is more complexity than the marginal coverage justifies.

```rust
fn arb_session_name() -> impl Strategy<Value = String> {
    prop_oneof![
        Just("-".to_string()),
        Just("Test Session".to_string()),
        "[A-Za-z0-9 _.\\-]{1,20}",  // bounded printable ASCII, no leading/trailing whitespace risk
    ]
}
```

The regex pattern uses a 1-20 length bound; proptest shrinks toward the empty match, but the `{1,20}` lower bound prevents a zero-length name (which would render as `s=` and re-parse as `s=-` — covered by static fixtures).

### 3.3 Connection (`c=`)

Session-level `c=` is always present in builder output. We do not vary that — the builder unconditionally emits one.

Per-media `c=` is only set if we mutate the `MediaDescription` after `.build()`. Strategy: `prop_oneof![Just(None), arb_ip().prop_map(|ip| Some(media_connection(ip)))]` shrinking toward `None`.

### 3.4 Timing (`t=`)

The oracle calls these "lossy timing values" for the non-numeric case. To stay in the lossless range, generate u64 numerals only:

```rust
fn arb_timing() -> impl Strategy<Value = (u64, u64)> {
    // Bound the range — u64::MAX would render as a 20-digit number that
    // makes failure messages unreadable. The round-trip behavior is
    // identical for any in-range u64 that parses cleanly.
    (0u64..=2_000_000_000, 0u64..=2_000_000_000)
}
```

(2e9 covers the standard NTP-style timestamp space comfortably; nothing in the oracle cares which exact value we pick.)

### 3.5 MediaDescription (`m=`)

```rust
fn arb_media_type() -> impl Strategy<Value = MediaType> {
    // MediaType::Other is a known lossy collapse: input "m=image ..."
    // re-serializes as "m=other ...". The oracle absorbs this on the
    // first round-trip. To stay in the *lossless* subset we exclude
    // ::Other from the generator. Static fixture rt_sdp_media_type_other
    // already covers the lossy path.
    prop_oneof![
        Just(MediaType::Audio),
        Just(MediaType::Video),
        Just(MediaType::Application),
        Just(MediaType::Message),
    ]
}

fn arb_protocol() -> impl Strategy<Value = String> {
    prop_oneof![
        Just("RTP/AVP".to_string()),
        Just("RTP/SAVP".to_string()),
        Just("RTP/AVPF".to_string()),
        Just("RTP/SAVPF".to_string()),
    ]
}

fn arb_payload_type() -> impl Strategy<Value = u8> {
    // Static (0..=34) and dynamic (96..=127) PT ranges per RFC 3551.
    // Excluding the reserved/unassigned ranges keeps generated PTs
    // semantically sane even though the parser doesn't enforce it.
    prop_oneof![0u8..=34, 96u8..=127]
}

fn arb_port() -> impl Strategy<Value = u16> {
    // Even ports for RTP are conventional but not required. We allow
    // any u16 — port 0 is the "rejected media" sentinel which the
    // parser/serializer handle losslessly (is_rejected() reads it back).
    any::<u16>()
}
```

`num_ports` is **NOT** generated. The HLD already documents that `num_ports` is silently dropped on serialize (a known semantic bug, fixed point still holds at s2). Generating it via builder mutation would force the test through that drop, which is a static-fixture concern (`rt_sdp_num_ports`) not a property concern.

### 3.6 Bandwidth (`b=`)

`MediaDescription::bandwidth` is a `HashMap<String, u32>`. The serializer was recently fixed to sort by key (Stage 1 of the round-trip oracle JRN). We must:

1. Generate from a small, **distinct** set of well-known bandwidth modifiers (`AS`, `CT`, `RR`, `RS`, `TIAS`).
2. Generate values strictly inside the lossless u32 numeric range.
3. Not rely on insertion order — we collect `Vec<(String, u32)>` and `Iterator::collect` into the HashMap.

```rust
fn arb_bandwidth_modifier() -> impl Strategy<Value = String> {
    prop_oneof![
        Just("AS".to_string()),
        Just("CT".to_string()),
        Just("RR".to_string()),
        Just("RS".to_string()),
        Just("TIAS".to_string()),
    ]
}

fn arb_bandwidth() -> impl Strategy<Value = HashMap<String, u32>> {
    // Up to 5 entries (one per modifier above). prop::collection::hash_map
    // dedupes on key automatically.
    prop::collection::hash_map(arb_bandwidth_modifier(), 0u32..=10_000_000, 0..=5)
}
```

We deliberately do **not** generate session-level bandwidth — the parser drops it entirely (the HLD's "Session-level `b=` is dropped entirely" entry). Generating it would force the test through a normalization, which is the static-fixture corpus's job.

### 3.7 Attributes (`a=`)

The `Attribute` struct is `{ name: String, value: Option<String> }` and the parser/serializer round-trips it as `a=name` or `a=name:value` literally (no semantic interpretation). To stay in the lossless subset we generate from a curated allowlist:

```rust
fn arb_direction() -> impl Strategy<Value = Direction> {
    prop_oneof![
        Just(Direction::SendRecv),
        Just(Direction::SendOnly),
        Just(Direction::RecvOnly),
        Just(Direction::Inactive),
    ]
}

fn arb_rtpmap_attr(pt: u8) -> impl Strategy<Value = Attribute> {
    // Curated codec list. Each combination renders as
    //   "<pt> <encoding>/<clock_rate>" or
    //   "<pt> <encoding>/<clock_rate>/<channels>"
    // and re-parses identically.
    prop_oneof![
        Just(("PCMU", 8000u32, None)),
        Just(("PCMA", 8000u32, None)),
        Just(("G722", 8000u32, None)),
        Just(("opus", 48000u32, Some(2u8))),
        Just(("telephone-event", 8000u32, None)),
    ].prop_map(move |(enc, rate, ch)| {
        let value = match ch {
            Some(c) => format!("{} {}/{}/{}", pt, enc, rate, c),
            None    => format!("{} {}/{}", pt, enc, rate),
        };
        Attribute { name: "rtpmap".to_string(), value: Some(value) }
    })
}

fn arb_fmtp_attr(pt: u8) -> impl Strategy<Value = Attribute> {
    // params has no internal whitespace handling beyond the "<pt> <params>"
    // split, so we constrain to a small set of safe payloads.
    prop_oneof![
        Just("0-16".to_string()),
        Just("useinbandfec=1".to_string()),
        Just("mode-set=0,2,4".to_string()),
    ].prop_map(move |params| Attribute {
        name: "fmtp".to_string(),
        value: Some(format!("{} {}", pt, params)),
    })
}

fn arb_ice_attrs() -> impl Strategy<Value = Vec<Attribute>> {
    // Optional block — Some(_) emits ufrag+pwd; None emits nothing.
    // We do NOT vary candidate-line space here; ice_attrs.rs already
    // unit-tests Candidate::to_sdp / from_sdp.
    prop_oneof![
        Just(Vec::<Attribute>::new()),
        ("[A-Za-z0-9]{4,8}", "[A-Za-z0-9]{22,40}").prop_map(|(u, p)| vec![
            Attribute { name: "ice-ufrag".into(), value: Some(u) },
            Attribute { name: "ice-pwd".into(),   value: Some(p) },
        ]),
    ]
}

fn arb_other_attr() -> impl Strategy<Value = Attribute> {
    // See §9 open question. For now, restrict to a hand-picked set
    // of attribute names known NOT to overlap with anything the
    // parser or any helper interprets specially.
    prop_oneof![
        Just(Attribute { name: "ptime".into(),    value: Some("20".into()) }),
        Just(Attribute { name: "maxptime".into(), value: Some("40".into()) }),
        Just(Attribute { name: "rtcp-mux".into(), value: None }),
        Just(Attribute { name: "label".into(),    value: Some("audio0".into()) }),
    ]
}
```

### 3.8 Composition

```rust
fn arb_media_builder() -> impl Strategy<Value = MediaBuilder> {
    (
        arb_port(),
        arb_protocol(),
        arb_direction(),
        prop::collection::vec(arb_payload_type(), 1..=4),  // ≥1: m= line requires ≥1 fmt
    ).prop_flat_map(|(port, proto, dir, pts)| {
        // For each PT, attach an rtpmap (mandatory for dynamic PTs;
        // safe for static PTs). For some PTs, also attach an fmtp.
        // Build via MediaBuilder::audio(port).protocol(...).direction(...)
        // and per-PT codec()/fmtp() chained calls.
        ...
    })
}

fn arb_session_description() -> impl Strategy<Value = SessionDescription> {
    (
        arb_origin_username(),
        arb_numeric_string(u64::MAX),    // session_id
        arb_numeric_string(u64::MAX),    // session_version
        arb_ip(),                        // local_addr (drives net/addr type)
        arb_session_name(),
        arb_timing(),
        prop::collection::vec(arb_media_builder(), 0..=3),
        arb_ice_attrs(),                 // attached to first media if present
    ).prop_map(|(user, sid, sver, addr, name, (start, stop), media, ice)| {
        let mut sdp = SdpBuilder::new(addr)
            .username(user)
            .session_id(sid.parse().unwrap_or(0))
            .session_version(sver.parse().unwrap_or(0))
            .session_name(name);
        for mb in media { sdp = sdp.add_media(mb); }
        let mut built = sdp.build();
        built.timing = Timing { start, stop };
        // Path-(b) escape hatch: attach generated bandwidth, ICE attrs,
        // optional media-level connection.
        if let Some(first) = built.media.first_mut() {
            first.attributes.extend(ice);
        }
        built
    })
}
```

## 4. Oracle

```rust
let bytes = sdp.to_string().into_bytes();
oracle::assert_roundtrip_fixed_point(&bytes);
```

We do **not** invent a separate oracle. The shared oracle's contract (parse-fail → silent return; first round-trip may normalize; s2 == s3 thereafter) is exactly what we want. A generated SDP that fails to parse is silently no-op'd by the oracle, which is fine — our generators are designed to always parse, and any case that doesn't is itself a finding worth investigating (caught by an additional sanity assertion):

```rust
proptest! {
    #![proptest_config(ProptestConfig::with_cases(256))]
    #[test]
    fn proptest_sdp_session_roundtrip(sdp in arb_session_description()) {
        let bytes = sdp.to_string().into_bytes();
        // Sanity: generator must always produce parseable SDP.
        // Without this, an oracle no-op silently passes the case.
        prop_assert!(
            SessionDescription::parse(std::str::from_utf8(&bytes).unwrap()).is_ok(),
            "generator produced unparseable SDP: {}",
            String::from_utf8_lossy(&bytes),
        );
        oracle::assert_roundtrip_fixed_point(&bytes);
    }
}
```

The `prop_assert!` is the key difference vs. the libfuzzer driver: there, parse-fail is the common case; here, parse-fail is a generator bug we want to surface.

## 5. Shrinking

A failure shrinks toward the **minimum viable SDP per RFC 4566**: `v=0`, `o=- 0 0 IN IP4 0.0.0.0`, `s=-`, `t=0 0`, with no media. This is the smallest input the parser accepts (verified by `tests/fixtures/sdp/minimal.sdp` and the `MissingVersion`/`MissingOrigin`/`MissingTiming` parser tests).

How we ensure this:

- **No `prop_filter` past the minimum-viable boundary.** `prop_filter` rejects shrunk values, slowing convergence; we use composite strategies so every shrink step is constructively valid by definition.
- **Each `prop_oneof!` arm is ordered "simplest first".** Proptest shrinks toward the first arm. So:
  - `arb_session_name` shrinks toward `Just("-")` (the default).
  - `arb_direction` shrinks toward `Just(Direction::SendRecv)` (the default).
  - `arb_media_type` shrinks toward `Just(MediaType::Audio)` (the most-tested type).
  - `arb_ip` shrinks toward `0.0.0.0`.
  - `arb_ice_attrs` shrinks toward `Just(Vec::new())` (no ICE attrs).
- **Numeric ranges shrink toward 0.** `0u64..=2_000_000_000` shrinks to 0 (legal `t=0 0` per RFC 4566 §5.9).
- **Vec lengths shrink toward 0.** `prop::collection::vec(_, 0..=3)` shrinks toward `vec![]` — zero media descriptions, which is the minimum-viable shape.
- **`prop::collection::vec(_, 1..=4)` for formats** has a lower bound of 1 because an `m=` line with zero formats fails parse (`MediaDescription::parse` requires `parts.len() >= 4`). This is the only place we have a hard non-zero floor in a vec; it is intentional and shrinks to length-1, not zero.

A shrunk failure should look something like:

```
v=0
o=- 0 0 IN IP4 0.0.0.0
s=-
c=IN IP4 0.0.0.0
t=0 0
m=audio 0 RTP/AVP 0
a=rtpmap:0 PCMU/8000
a=sendrecv
```

— a single audio media with one static-PT codec and no extras. Anything more complex than that means the failure depends on something more than the minimum viable shape, which itself is useful triage.

## 7. Run shape

`ProptestConfig::cases = 256` (proptest's default).

Justification for not exceeding 256:
- The oracle invocation is the heavy step; each run does parse + serialize + parse + serialize + parse. On a generated 5-media SDP that's ~5 × ~10 μs = ~50 μs per oracle call → 256 × 50 μs = ~13 ms total. Well under any test-suite budget.
- The SDP space is wider than (e.g.) a single SIP header, but it's not so wide that 1024 cases would surface bugs 256 wouldn't. The marginal coverage gain past 256 is approximately the marginal noise — better invested in extending the strategy tree if a real gap emerges.
- If a finding lands and we want to harden the regression test, we can either (a) bump `cases` for that one test, or (b) freeze the failing input as a static fixture and rely on the existing static-fixture driver. Option (b) is the convention — it's deterministic, fast, and named.

`ProptestConfig::max_shrink_iters = 1024` (proptest default). No reason to deviate.

## 9. Open questions

### 9.1 `Attribute::Other` re-recognition risk (PRIMARY OPEN QUESTION)

The `Attribute` struct is opaque to the parser — `Attribute::parse` just splits on the first `:`. But many *consumers* of `MediaDescription::attributes` (`MediaDescription::direction`, `rtpmaps`, `fmtps`, `ice_attrs::read_*`) match on `attr.name == "rtpmap"` / `"sendrecv"` / `"ice-ufrag"` / etc. The round-trip oracle compares `SessionDescription` ASTs, and `Attribute: PartialEq` is a literal name+value compare, so this should be safe — but consider the case:

- Generator emits `Attribute { name: "rtpmap", value: Some("not actually a rtpmap") }` in the `arb_other_attr` channel.
- The serializer renders `a=rtpmap:not actually a rtpmap`.
- The parser's `Attribute::parse` splits on `:` and stores `Attribute { name: "rtpmap", value: Some("not actually a rtpmap") }` — exactly what we put in.

So at the AST level we're fine, *but* if any future change makes the parser typed (e.g., reject malformed rtpmap values at parse time), our generator would suddenly start producing inputs that fail the first parse. That would be a silent oracle no-op, undetectable.

**Question for the reviewer:** should `arb_other_attr` enforce a denylist of names that have typed semantics elsewhere in the codebase (the union of `rtpmap`, `fmtp`, `sendrecv`, `sendonly`, `recvonly`, `inactive`, `ice-ufrag`, `ice-pwd`, `candidate`, `rtcp-mux`, `ptime`, `maxptime`, `group`, `mid`, `setup`, `fingerprint`, `extmap`, `ssrc`, `rtcp`, `crypto`, `tls-id`, `direction`, …)? Or is the AST-level equality guarantee sufficient and we treat the "future typed parser" risk as the future-change's problem?

The current proposal in §3.7 sidesteps this by using a tiny hand-picked list (`ptime`, `maxptime`, `rtcp-mux`, `label`) — which is conservative but loses generator coverage on the long tail of legitimately-untyped attributes. A more aggressive design would generate `name` from a regex with a denylist filter; but `prop_filter` slows shrinking, which we want to avoid (§5).

### 9.2 Should we generate ASCII-only or full UTF-8 in string fields?

The parser uses `for line in sdp.lines()` and per-line `.trim()`, which are UTF-8-safe by construction (verified by the `test_parse_non_ascii_first_char_fuzz_001` regression test in `parser.rs`). The serializer uses `writeln!`, also UTF-8-safe. So in principle we could generate `\p{L}` and friends in `session_name`, `username`, etc.

In practice, generating non-ASCII would expand the failure-message space (UTF-8 multibyte chars render as `\u{...}` escapes in proptest output, which is hard to read) without exercising any new code path. Recommendation: ASCII-only at the regex level (`[A-Za-z0-9 _.\-]`). The fuzz target already covers the non-ASCII path.

### 9.3 Should `arb_ip6` include the `2001:db8::1`-with-zone-id form (`fe80::1%eth0`)?

The parser stores `unicast_address` and `connection.address` as plain strings, so it does not canonicalize. `Ipv6Addr::to_string` does not emit a zone id. If a generator passed `IpAddr::V6` through the builder, no zone id appears. We do not need to generate this. Out of scope.

## 10. Acknowledged trade-offs

- **No "first round-trip is bytes-equal" assertion.** The oracle's contract is that the *second* round-trip is the fixed point. We do not strengthen it for proptest — the static-fixture driver's `rt_sdp_oracle_holds_on_canonical_input` already exercises the s1 == s2 case for canonical inputs, and our generator does not produce canonical inputs deterministically (e.g., `Origin::session_id` rendered from a `String` that came from a `u64`-to-string is canonical, but the builder also emits a session-level `c=` that the round-trip preserves; we'd need to constrain the generator further to guarantee s1 == s2).
- **Single test, no per-feature property breakdown.** We do not split into "property: bandwidth round-trips", "property: rtpmap round-trips" etc. The single property "any builder-constructible SDP round-trips at s2" subsumes all of them. Sub-properties become valuable only when one fails and we want a regression-pin — at that point, a static fixture is simpler.
- **256 cases × ~50 μs ≈ 13 ms is fast enough that we run the property in the standard `cargo test` path, not behind a feature gate.** No `#[cfg(feature = "proptest")]` or env-var gating.
