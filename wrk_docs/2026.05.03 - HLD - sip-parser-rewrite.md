# Motivation

`rsiprtp` claims to be a SIP/RTP stack rewrite. Today the parser is the
third-party crate `rsip` 0.4. Everything else — transactions, dialogs, auth,
transport, RTP, SRTP, ICE, media, codecs, session timers — is ours. The
parser is the most "SIP-shaped" piece of any SIP stack and the first thing
a SIP person opens; depending on someone else's parser undercuts the
"complete rewrite" framing. Owning it makes the story honest.

**No crates.io publication yet** (placeholder reserved at v0.0.0; first
real release pending). Public API shape is not committed; we can redesign
freely without compat tax. This is the right window.

Secondary drivers:

- **`rsip` 0.4 is showing its age.** It drags `syn 1`, `digest 0.9`,
  `sha2 0.9`, `uuid 0.8` as transitive deps. `cargo-deny` already flags
  the duplicate-version warnings.
- **Bus factor.** `rsip` 0.5 unpublished, maintainer effectively solo.
- **`ftth-rsip 0.4.1`** (`menhera-org/rsip`, MIT) is a maintained fork.
  It's the obvious **Plan B** if the rewrite slips: ~half-day swap, likely
  resolves duplicate-version noise. Doesn't satisfy "we wrote the stack";
  use only as pressure-release valve.

# Scope

**In scope:**

- Drop `rsip` from runtime dependencies of `crates/rsiprtp`.
- Implement an in-tree parser at `crates/rsiprtp/src/sip/parser/` covering
  request/response framing, headers, URIs, typed-form structs.
- **Redesign the public API of `crate::sip`** to remove rsip-typed return
  values. Pre-1.0, unpublished, no compat tax — this is the moment.
- Keep `rsip` in `[dev-dependencies]` indefinitely as the differential-test
  oracle.

**Out of scope:**

- Any non-parser change to `crate::transaction`, `crate::dialog`,
  `crate::transport`, `crate::session`, `crate::rtp`, etc. — these get
  *touched* (call-site updates for the new return types) but not redesigned.
- `crate::sip::auth` — already ours, no rsip.

**Non-goals:**

- Zero-copy parsing. Owned `String`s like rsip and mdsiprtp3 for the
  first cut. Revisit at 0.5.0+.
- Streaming parser. UDP datagrams are whole; TCP layer buffers a complete
  message before parsing.

# Surface — what `rsip` we actually use

Audited 2026-05-03 against `crates/rsiprtp/src/`. Three categories:

## Public API leakage (~10 functions)

These return rsip types from `crate::sip`. All change in this rewrite:

```
SipRequest::uri()                → returns &rsip::Uri
SipRequest::from_uri()           → returns Result<rsip::Uri>
SipRequest::to_uri()             → returns Result<rsip::Uri>
SipRequest::contact_uri()        → returns Option<rsip::Uri>
SipRequest::from_tag_and_uri()   → returns Result<(String, rsip::Uri)>
SipRequest::inner()              → returns &rsip::Request   ← escape hatch
SipResponse::contact_uri()       → returns Option<rsip::Uri>
SipResponse::inner()             → returns &rsip::Response  ← escape hatch
Method::to_rsip()                → returns rsip::Method
Method::from(&rsip::Method)      → From impl
pub use rsip::Uri as RsipUri     → public re-export
```

`SipRequest::inner()` and `SipResponse::inner()` have **zero callers
inside rsiprtp**. They exist only as escape hatches; remove outright.

`Method::to_rsip()` and `Method::from(&rsip::Method)` only make sense as a
bridge while rsip is in the runtime; remove with rsip.

The four `*_uri()` accessors get retyped to return `&SipUri` /
`Option<SipUri>` / `Result<SipUri>`. **Internal call sites mostly do
`.to_string()` immediately** — only `session/call.rs:207-208, 242-243`
and a handful of test sites consume the URI value, all via Display. Low
real churn behind the apparent breakage.

# Architectural principle — adopt rsip's two-tier model

rsip's design (per its lib.rs docs):

> by default rsip will parse all headers and convert them to distinct
> untyped header types ... Each untyped header is basically a NewType
> around String. ... any parsing on complex headers takes place only
> when needed, on demand.

This is exactly the right model and **we keep it**:

1. **Tier 1 — Framing + header recognition.** Eager. Splits the message
   into request/status line + ordered header list + body. Each header
   becomes a typed enum variant with the body still as `String`.
2. **Tier 2 — Typed parsing.** Lazy. `.typed()` on a header parses the
   string into structured form (`NameAddr`, `CSeq`, `Via`, etc.). Cost
   paid only when the consumer asks.

Why this matters:

- Robustness: a malformed exotic header doesn't fail message parse.
- Performance: hot-path consumers (transactions, routing) only touch
  Call-ID, CSeq, Via — they never pay for parsing CallInfo or Warning.
- Forward-compat: unknown headers ride through as `Header::Other` with no
  loss.

# Header enum delta — rsip 47 vs ours 19 + Other

rsip's `Header` enum has ~47 variants. We have 19 typed variants plus
`Other`. The remaining ~28 (`Accept`, `AcceptEncoding`, `AcceptLanguage`,
`AlertInfo`, `AuthenticationInfo`, `CallInfo`, `ContentDisposition`,
`ContentEncoding`,
`ContentLanguage`, `Date`, `ErrorInfo`, `Event`, `InReplyTo`,
`MimeVersion`, `MinExpires`, `Organization`, `Priority`, `ProxyRequire`,
`ReplyTo`, `RetryAfter`, `Server`, `Subject`, `SubscriptionState`,
`Timestamp`, `Unsupported`, `UserAgent`, `Warning`) ride rsip's typed
channel today.

In our parser those 28 fall into `Header::Other`. **The diff-test harness
must normalize this** — a `Server: foo` header parses as
`Header::Server(...)` in rsip and `Header::Other("Server", "foo")` in
ours. Both are correct for the data they carry; the diff harness
collapses both forms to the neutral `(name_lowercase, value_normalized)`
representation before comparison.

Decision: **don't add native variants for the 26 unused headers in this
rewrite.** Keep them as `Other` until a real consumer needs typing.
Premature variants = premature design. Easy to add later.

# What carries over from `mdsiprtp3`

`mdsiprtp3/src/sip/` (~1,400 lines, no rsip, hand-rolled splits, **101
existing `#[test]` functions across the module** — substantial seed for
the diff harness corpus) is the structural seed.

| mdsiprtp3 file | lines | role | reuse strategy |
|---|---|---|---|
| `method.rs` | 118 | `Method` enum + parse/display | **Adopt + extend.** Add `Publish` (mdsiprtp3 missing it; rsip has it). Make `from_str` case-insensitive (mdsiprtp3 is case-sensitive; rsip is not; RFC 3261 permits CI). Naming: `Prack` (matches our public API) not `PRack` (rsip's form). |
| `status.rs` | 191 | `StatusCode` u16 wrapper + reason phrases | **Adopt as-is.** Already complete. |
| `uri.rs` | 242 | URI parser sip/sips/tel | **Adopt + harden.** Lacks IPv6 references (`[::1]:5060`) and quoted-param. Cross-merge with our existing `crate::sip::uri::SipUri` (which already handles IPv6 brackets). |
| `headers.rs` | 382 | `Headers` HashMap + `NameAddr`, `CSeq`, `Via` | **Adopt structure, replace HashMap with order-preserving Vec.** rsip's `Headers` is `Vec<Header>`-backed and our wire-format round-trip relies on order. mdsiprtp3's HashMap loses it — fundamental flaw. |
| `via.rs` | 215 | NAT-aware `ViaHeader` | **Merge with our existing `crate::sip::headers::Via`.** Ours already covers rport/received; discard the duplicate. |
| `message.rs` | 427 | `SipMessage` + `Request` + `Response` framing, header parsing with folding | **Adopt framing; rewrite typed accessors against new wrapper API.** |
| `dialog/transaction/auth/ua/uac/uas/register` | ~1,800 | UA-stack sketches | **Discard.** rsiprtp's versions are RFC-complete and battle-tested. |

Net seed: ~1,200 lines of mdsiprtp3 structure + 101 tests.

**rsip's source is MIT** (Filippos Vasilakis). We can read freely and copy
with attribution where useful. The reason to write from spec is *our*
understanding, not licensing. Treat rsip as a reference, mdsiprtp3 as the
seed, RFCs as truth.

# What needs writing fresh

1. **RFC 3261 ABNF compliance for header values.** mdsiprtp3 splits on
   delimiters; spec requires honoring quoted strings, escape sequences,
   comments in `(...)`. Re-use the seed's structure with stricter
   value-level parsing.
2. **Header folding correctness + size limits.** mdsiprtp2 patterns
   (`MAX_HEADER_VALUE_LEN`, `MAX_START_LINE_LEN`, `MAX_HEADERS`) carried
   forward as defense-in-depth.
3. **Native `Header` enum** with the 19 typed variants we currently
   use, plus `Other(String, String)`. Each variant holds the raw value
   as `String` for tier-1, with a tier-2 `.typed()` method where
   applicable.
4. **Typed-form structs** `From`, `To`, `Via`, `CSeq`, `Contact` with the
   exact accessors today's `rsip::typed::*` provide (`.tag()`, `.uri`,
   `.branch()`, `.seq`, `.method`).
5. **`Headers` collection** preserving wire order. `Vec<Header>` with
   index helpers, **not HashMap**.
6. **Wire-format serializer.** Canonical form: long header names,
   single-space after colon, `\r\n` line endings, body separator
   `\r\n\r\n`.
7. **Parser entry points** `SipMessage::parse(&[u8])`,
   `Request::parse`, `Response::parse`, with header-folding-aware framing.
8. **Differential test harness** (separate section).

# Differential-test harness

Two parsers, one corpus, asserted equivalence. The unlock pattern.

```
                      ┌────────────────┐
   raw SIP bytes ───→ │ rsip 0.4       │ ──→ DiffMessage A
        │             │ (dev-dep)      │
        │             └────────────────┘
        │
        │             ┌────────────────┐
        └───────────→ │ crate::sip     │ ──→ DiffMessage B
                      │ (new parser)   │
                      └────────────────┘
                                     │
                              assert A ≡ B
```

**Implementation:**

1. `crates/rsiprtp/tests/parser_diff.rs` (new). One harness fn:
   `fn assert_equivalent(bytes: &[u8])`.
2. **Neutral `DiffMessage` representation:**
   - `kind: Request|Response`
   - `method: String` / `status: u16`
   - `uri: NormalizedUri` (lowercased scheme/host, parameters
     order-independent)
   - `headers: Vec<(name_lc, value_normalized)>` — order preserved
   - `body: Vec<u8>` — verbatim
3. **Header normalization**: header names lowercased; compact forms
   resolved to long form; values whitespace-collapsed (multiple spaces →
   one); comments `(...)` stripped; native rsip-typed variants and our
   `Other` collapsed to `(name, value)` form so the 26-header gap doesn't
   produce false divergences.
4. **Equivalence criterion** (tighter than rev 1):
   - Both accept ⇒ DiffMessage must be equal.
   - Both reject ⇒ no further check (errors may differ).
   - One accepts, one rejects ⇒ **investigate**, do not silently allow.
     This is where real RFC-interpretation bugs hide. Triage each case:
     spec is explicit (fix the wrong one), spec is ambiguous (document
     the choice), wire-real-world says one thing (follow it).
5. **Corpora** (in priority order):
   - **mdsiprtp3's 101 test inputs** — extract from `#[test]` functions
     into `tests/fixtures/mdsiprtp3/`. Free seed corpus.
   - **rsiprtp's existing fuzz corpus** at `crates/rsiprtp/fuzz/corpus/`.
   - **Hand-curated golden set** from `src/sip/tests/rfc3261.rs`.
   - **RFC 4475 torture tests** — RFC body is public domain, messages
     copy-pastable. New fixture dir.
   - **Live capture** from baresip integration tests (optional, later).
6. **Fuzz target augmentation**: `sip_message_parse_diff` runs both
   parsers and panics on divergence. Same overnight cadence as today's
   fuzz campaign.

**Acceptance gate**: zero divergences across the union of corpora before
the new parser becomes the runtime parser.

# Phasing

Diff-testing comes online **at milestone 2**, not 4 — earlier we run
both parsers, earlier we find bugs.

| # | milestone | exit gate |
|---|---|---|
| 1 | `parser/` skeleton + `Method` (seeded from mdsiprtp3, +Publish, CI from_str) + `StatusCode` | `cargo test -p rsiprtp` green; new types compile, unused |
| 2 | **Diff-test harness online (early).** `Header` enum + ordered `Headers` collection + framing parser. `parser_diff.rs` runs Tier-1 only against the mdsiprtp3 fixtures and rsiprtp fuzz corpus | Zero divergences on Tier-1 (header *count* + names + raw values); Tier-2 typed forms not yet involved |
| 3 | URI parser (absorb `SipUri` + mdsiprtp3 features) | `SipUri` tests still pass; URI normalized in DiffMessage |

| 7 | **Public-API redesign**: `*_uri()` returns `SipUri`; remove `inner()`, `Method::to_rsip`, `Method::from(&rsip)`, `RsipUri`. Update internal call sites | All callers compile; full_test bar green; `cargo public-api` shows zero `rsip::` leakage |
| 8 | Cut over `crate::sip::message.rs` to use new parser internally | `cargo test -p rsiprtp` green; rsip still in dev-deps |

| 12 | CHANGELOG, README update, version bump to 0.4.0 | Ready to publish |

Each milestone is a single PR-sized step. **Realistic effort: 6-8 focused
weeks** (revised up from 4-6 in rev 1; the public-API redesign milestone
alone is a week).

**M10 follow-up note**: when rsip is moved out of the runtime deps and
our parser becomes authoritative, the following deferred-re-enable
tests in `crates/rsiprtp/tests/parser_diff.rs` should be retargeted to
direct on-our-parser assertions (no rsip-side comparison). Each is a
real rsip 0.4 spec deficiency; our parser correctly handles each:

1. `diff_handcrafted_invite_folded_subject` (`#[ignore]`d) — rsip 0.4
   rejects RFC 3261 §7.3.1 line folding.
2. `typed_from_quoted_param_value_rsip_rejects_broadly` — rsip 0.4
   rejects all quoted-string param values (RFC 3261 §25.1 `gen-value`).
3. `typed_from_quoted_param_value_with_semicolon_rsip_rejects` — subset
   of #2 with embedded `;`.
4. `typed_via_ipv6_rsip_rejects` — rsip rejects IPv6 in Via sent-by
   (RFC 5118 §4.1).

**M8 entry condition**: `parser::typed::Contact` is an enum (`Wildcard |
Addr`), not a struct like `rsip::typed::Contact`. The cutover at M8 will
match on the variant rather than dot-accessing `.uri` directly. This is
deliberate — wildcard is real wire reality (REGISTER unbinds, RFC 3261
§10.3); forcing callers to handle it explicitly beats forgettable
`is_wildcard()` flags.

**M11 expectations**: with ~10 fixtures the harness has already surfaced
4 rsip 0.4 spec deficiencies. M11's overnight fuzz against the same
harness should anticipate a high volume of rsip-side rejection
divergences; the operator is triaging and pinning rsip-side bugs per
the M5 pattern, not chasing our-parser bugs.

# Performance threshold

Bench `parser_diff` corpus parse-throughput before/after cutover.

- **Acceptable for first cut:** ≤2× rsip parse time on representative
  messages.
- **Investigate but not block:** 2-5× slower (likely a hot-path
  allocation; profile, fix in scope).
- **Block cutover:** >5× slower (architectural problem, not just polish).

Zero-copy is a 0.5.0+ follow-up, explicitly punted from this rewrite.

# Risks

| risk | mitigation |
|---|---|
| RFC 3261 grammar edge cases bite us months later | RFC 4475 torture tests as a hard gate; fuzz with rsip as oracle indefinitely |
| Performance regression vs rsip | Benchmark before cutover; thresholds above |
| Public-API churn surprises downstream consumers | None today (no published version, no consumers). 0.4.0 release captures all changes in one CHANGELOG entry |
| Effort balloon | Per-milestone-mergeable. Worst case: ship milestones 1-7 as `0.4.0-alpha`, continue. Plan B: `ftth-rsip` swap if pressure mounts |
| Maintenance of two parsers in dev-deps forever | dev-dep is fine. If `rsip` truly dies, dev-dep stays pinned and runtime is unaffected |
| Header enum delta produces diff-test false positives | Normalization layer in DiffMessage explicitly flattens rsip's 47 variants to our 19 typed + Other |
| `mdsiprtp3` seed encodes naive splits we then have to debug | Seed is *structural* (file layout, type shapes); parsers rewritten value-by-value with property tests + diff-test catching divergences early |
| One-accepts-one-rejects divergences during diff-test | Triage protocol in the harness section (don't silently normalize away — investigate each) |

# Exit criteria

The rewrite is "done" when:

1. `cargo tree -p rsiprtp -e normal` shows no `rsip` in runtime deps.
   `rsip` may remain only under `[dev-dependencies]`.
2. `cargo public-api -p rsiprtp` shows zero `rsip::` references.
3. `crates/rsiprtp/tests/parser_diff.rs` runs zero divergences across:
   - mdsiprtp3 fixtures
   - hand-curated golden cases from `tests/rfc3261.rs`
   - RFC 4475 torture-test fixtures
   - existing fuzz corpus
4. `sip_message_parse_diff` fuzz target runs ≥8 hours overnight with zero
   divergences.
5. full_test bar (fmt, clippy, deny, build, tests, doc, llvm-cov) green.
   Coverage of new `parser/` module ≥95% lines (lower than crate-wide 99%
   because adversarial-input branches are hard to deterministically
   cover; explicit `#[cfg(coverage_nightly, coverage(off))]` only with
   justification per the existing coverage discipline).
6. CHANGELOG `[Unreleased]` entry covers Removed (`RsipUri`, `inner()`,
   `Method::to_rsip`, `Method::from(rsip::Method)`), Changed (`*_uri()`
   return types), Added (in-tree parser, RFC 4475 fixtures).
7. README "Signaling" section adds: *"SIP message parsing and building —
   in-tree implementation, no third-party SIP parser dependency."*
8. Version bumped to **0.4.0** for the public-API redesign.

# Plan B — switch to `ftth-rsip`

If the rewrite slips or another priority intervenes:

1. Cargo.toml: `rsip = "0.4"` → `ftth-rsip = "0.4"` (use renaming if API
   compatible).
2. Smoke-test workspace; expect a few API drift fixes.
3. Delete this HLD; flag duplicate-version warnings as "resolved by
   maintained fork."

Cost: half a day. Value: removes the maintenance-bus-factor risk without
the rewrite. Doesn't satisfy "we wrote the SIP stack." Only invoke if
schedule pressure dominates story pressure.
