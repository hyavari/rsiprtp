# Ralph Prompt: Gabby Voice AI Agent

## Goal

Build **gabby**, a voice AI SIP application that accepts incoming phone calls and enables natural conversation using Vosk (STT), Ollama (LLM), and Piper (TTS).

## Context

- This is the mdsiprtp SIP/RTP stack repository
- Gabby should be a separate binary crate at `crates/gabby/`
- Reference existing examples: `crates/mdsiprtp/examples/ai_bridge.rs` and `voicemail.rs`
- The stack uses sans-IO patterns with tokio for async I/O

### Phase 4: SIP/RTP Server

11. **Create `crates/gabby/src/server.rs`** - Main server:
```rust
pub struct GabbyServer {
    config: GabbyConfig,
    vosk_model: Arc<vosk::Model>,
    sip_socket: UdpSocket,
    calls: HashMap<String, CallHandle>,
}

impl GabbyServer {
    pub async fn new(config: GabbyConfig) -> Result<Self>;
    pub async fn run(self) -> Result<()>;
}
```
   - Bind UDP socket for SIP
   - Load Vosk model once (shared across calls)
   - Accept INVITE requests
   - Spawn call handler task per call
   - Handle BYE to terminate calls

12. **Create `crates/gabby/src/call.rs`** - Per-call handler:
```rust
pub struct CallHandler {
    call_id: String,
    remote_addr: SocketAddr,
    rtp_socket: UdpSocket,
    stt: SttProcessor,
    vad: VadState,
    llm: OllamaClient,
    tts: TtsProcessor,
    input_resampler: Resampler,  // 8k -> 16k
    output_resampler: Resampler, // 22k -> 8k
    output_buffer: VecDeque<i16>,
}

impl CallHandler {
    pub async fn new(...) -> Result<Self>;
    pub async fn run(mut self) -> Result<()>;
    async fn handle_audio(&mut self, samples: Vec<i16>);
    async fn process_turn(&mut self, transcript: &str);
    async fn send_greeting(&mut self);
}
```
   - Main loop: receive RTP, process audio, send RTP
   - Greeting on call start
   - STT → VAD → LLM → TTS pipeline
   - Stream TTS output as RTP packets (20ms frames)

## Technical Notes

### Audio Flow
- RTP uses G.711 (PCMU) at 8kHz, 20ms frames (160 samples)
- Vosk expects 16kHz mono PCM
- Piper outputs 22050Hz mono PCM
- Use rubato for high-quality resampling

### SIP Message Handling
- Parse INVITE, extract SDP for RTP port
- Send 200 OK with our SDP (PCMU codec)
- Handle ACK
- Handle BYE to end call
- Reference: `tests/stack_instance/mod.rs` for SIP message building

### Streaming Response
- Start TTS as soon as we have a complete sentence from LLM
- Buffer TTS output, send as 20ms RTP frames
- This reduces perceived latency significantly

### Error Handling
- If Vosk fails to load, exit with clear error
- If Ollama is unreachable, say "I'm having trouble thinking right now"
- If Piper fails, log error and skip that response

## Completion Criteria

When done, this should work:
1. `cd crates/gabby && ./scripts/setup.sh` downloads all models
2. Start Ollama: `ollama serve`
3. Run Gabby: `cargo run --release -p gabby`
4. Open Linphone, call `sip:gabby@localhost:5060`
5. Hear greeting: "Hello! I'm Gabby..."
6. Speak a question
7. Hear Gabby's response
8. Continue conversation naturally
9. Hang up to end
