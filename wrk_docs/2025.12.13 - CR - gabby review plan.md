# Gabby Voice AI Agent - Code Review Plan

## Overview

This document outlines the comprehensive code review plan for the `gabby` crate - a voice AI SIP agent that enables natural conversation over phone calls using STT (Vosk), LLM (Ollama), and TTS (Piper).

## Files to Review

### Core Application
1. `Cargo.toml` - Dependencies and build configuration
2. `src/main.rs` - Entry point, CLI parsing, server startup
3. `src/config.rs` - Configuration types and loading
4. `src/server.rs` - SIP server and call management
5. `src/call.rs` - Per-call handler and audio pipeline
6. `src/audio.rs` - Audio resampling and utilities

### Pipeline Components
7. `src/pipeline/mod.rs` - Module re-exports
8. `src/pipeline/stt.rs` - Vosk speech-to-text integration
9. `src/pipeline/vad.rs` - Voice activity detection
10. `src/pipeline/llm.rs` - Ollama LLM client
11. `src/pipeline/tts.rs` - Piper text-to-speech integration

### Scripts and Documentation
12. `scripts/setup.sh` - Dependency installation
13. `scripts/linphone_setup.md` - Softphone configuration
14. `README.md` - Project documentation
15. `gabby.example.toml` - Example configuration

## Review Categories

### 1. Correctness
- Logic errors and bugs
- Off-by-one errors
- Edge case handling
- API contract compliance (Vosk, Ollama, Piper)
- Audio sample rate handling
- RTP protocol compliance

### 2. Security
- Input validation (SIP messages, user speech)
- Command injection risks (subprocess spawning)
- Network exposure (binding addresses)
- Configuration file handling
- External service communication (Ollama API)

### 3. Error Handling
- Error propagation
- Graceful degradation
- Resource cleanup
- Recovery from transient failures
- Logging of errors

### 4. Performance
- Audio latency considerations
- Memory allocations in hot paths
- Buffer management
- Async operation efficiency
- Resampling efficiency

### 5. Code Quality
- Rust idioms and best practices
- Module organization
- API design
- Documentation completeness
- Test coverage

### 6. Robustness
- Concurrent call handling
- Resource exhaustion handling
- Timeout handling
- Shutdown behavior
- Signal handling

## Review Checklist Per File

For each file, evaluate:
- [ ] Purpose and responsibility clarity
- [ ] Error handling completeness
- [ ] Resource management
- [ ] Thread safety (where applicable)
- [ ] Input validation
- [ ] Edge cases
- [ ] Performance considerations
- [ ] Code style consistency
- [ ] Documentation adequacy

## Priority Areas

### High Priority
1. **server.rs** - SIP protocol handling, security exposure
2. **call.rs** - Audio pipeline correctness, resource management
3. **tts.rs** - Subprocess handling, command injection risk

### Medium Priority
4. **llm.rs** - External API integration, error handling
5. **stt.rs** - Audio processing correctness
6. **vad.rs** - Algorithm correctness
7. **audio.rs** - Resampling accuracy

### Lower Priority
8. **config.rs** - Configuration validation
9. **main.rs** - Application lifecycle
10. **Scripts and docs** - Completeness and accuracy
