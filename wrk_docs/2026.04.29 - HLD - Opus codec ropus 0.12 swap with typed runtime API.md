# HLD — Opus codec wrapper: bump to ropus 0.12, expose typed runtime API

## Why

The current `OpusCodec` wrapper was rewritten against `ropus 0.11`, which had no runtime mutation on the high-level `Encoder`. During that swap, four setters (`set_bitrate`, `set_fec`, `set_packet_loss`, `set_dtx`) were dropped because they had no real callers and no clean implementation path.

`ropus 0.12` adds a fully typed runtime mutation API on `Encoder`. With the rsiprtp stack still pre-publication, this is the moment to commit to the strongest shape rather than ship a simple-types wrapper and refactor later. Pre-publication = no API stability obligations, so the design choice is straightforward: pass the typed enums through.

## Goal

- Bump dep to `ropus = "0.12"`.
- Restore the four runtime setters as one-line delegations to the new high-level API, with typed enum signatures.
- Add symmetric getters for telemetry (REMB feedback, observability).
- Re-export the relevant ropus enums from `mdsiprtp-media` so callers do not depend on `ropus` directly.

## Non-goals

- Wiring `CongestionController` into the encoder. See the bridge briefing next to this file.
- Introducing a `Codec` / `AdaptiveBitrate` trait. That belongs to the bridge work where it gets a real caller.
- Migration code, deprecated-name shims, feature flags, env vars.

## Public API after the change

### Re-exports from `mdsiprtp-media`

```rust
pub use ropus::{Bitrate, InbandFec};
```

Exposed at the crate root and via `codec::opus` so existing import paths keep working. We re-export only the enums the wrapper's public API actually uses; `Signal`, `Bandwidth`, etc. stay inside `ropus` until a wrapper method needs them.

`Bitrate` variants: `Auto`, `Max`, `Bits(u32)`.
`InbandFec` variants: `Disabled`, `Enabled`, `Forced`.

Both enums derive `Debug, Clone, Copy, PartialEq, Eq, Hash` (verified in `ropus/src/api.rs`), so they work directly in tests with `assert_eq!`.

### `OpusConfig`

```rust
pub struct OpusConfig {
    pub sample_rate: u32,
    pub channels: u8,
    pub bitrate: Bitrate,   // was u32
    pub frame_ms: f32,
}
```

- `Default::default()` returns `Bitrate::Bits(32_000)`.
- `wideband_speech()` / `fullband_speech()` / `music()` use `Bitrate::Bits(_)` with their existing rates (24 / 32 / 96 kbps).

### `OpusCodec` — additions

```rust
// Setters: runtime-mutating, mid-stream safe per ropus 0.12.
pub fn set_bitrate(&mut self, b: Bitrate) -> Result<(), String>;
pub fn set_fec(&mut self, mode: InbandFec) -> Result<(), String>;
pub fn set_dtx(&mut self, on: bool) -> Result<(), String>;
pub fn set_packet_loss(&mut self, pct: u8) -> Result<(), String>;

// Getters: for telemetry and the future REMB feedback loop.
pub fn bitrate(&self) -> Bitrate;
pub fn effective_bitrate_bps(&self) -> i32;
pub fn inband_fec(&self) -> InbandFec;
pub fn dtx(&self) -> bool;
pub fn packet_loss_perc(&self) -> u8;
```

All setters / getters are direct delegations to the underlying `ropus::Encoder`. Setter errors are surfaced as `Result<(), String>` via `format!("...{e}")` to match the rest of `OpusCodec`'s error convention — `EncodeError` implements `Display` (via ropus's `impl_opus_error!` macro), so the existing wrapper's `{e:?}` formatting will be tightened to `{e}` at the same time. No new error type is introduced.

The existing surface (`new`, `with_config`, `name`, `sample_rate`, `channels`, `samples_per_frame`, `encode`, `decode`, `decode_plc`, `Default`) is unchanged.

## Implementation outline

Three files touched. No other crate references the changing types — re-verified: `OpusConfig` and `OpusCodec` are referenced only inside `crates/mdsiprtp-media`, and no call site reads `.bitrate` as a `u32`.

1. `Cargo.toml` (workspace) — `ropus = "0.11"` → `ropus = "0.12"`, and `version = "0.1.0"` → `version = "0.2.0"` (pre-1.0 breaking change to `OpusConfig.bitrate` field type, per CLAUDE.md versioning rule).
2. `crates/mdsiprtp-media/src/codec/opus.rs`:
   - Change `OpusConfig.bitrate` from `u32` to `Bitrate`. `Default` becomes `Bitrate::Bits(32_000)`; named constructors switch to `Bitrate::Bits(_)`.
   - Update `with_config` to pass `config.bitrate` straight through: `Encoder::builder(...).bitrate(config.bitrate).build()`.
   - Add the four setters and five getters as one-line delegations.
   - Tighten existing error formatting from `{e:?}` to `{e}`.
3. `crates/mdsiprtp-media/src/codec/mod.rs` and `crates/mdsiprtp-media/src/lib.rs` — re-export `Bitrate` and `InbandFec` so callers can `use mdsiprtp_media::Bitrate;`.

## Tests

In `codec/opus.rs`, restore the deleted tests against the new shapes and add a few that exercise the typed enums:

- `test_set_bitrate_explicit` — `Bitrate::Bits(24_000)` → encode → `Bitrate::Bits(64_000)` → encode; both calls return Ok, both encodes produce non-empty output, and `bitrate()` reflects the most recent value (`Bits(64_000)`).
- `test_set_bitrate_auto` — `Bitrate::Auto` setter returns Ok, `bitrate()` round-trips to `Bitrate::Auto`, and a subsequent encode succeeds. We do not assert anything about `effective_bitrate_bps()` numerically — the contract for *when* ropus populates that value is not part of what we are testing.
- `test_set_fec_modes` — round-trip each `InbandFec` variant (`Disabled`, `Enabled`, `Forced`) through the setter; verify via `inband_fec()` getter.
- `test_set_dtx_toggle` — set true / false; verify via `dtx()` getter.
- `test_set_packet_loss_perc` — set 10, 50; verify via `packet_loss_perc()` getter.
- `test_config_uses_bitrate_enum` — construct `OpusConfig { bitrate: Bitrate::Bits(48_000), .. }`, build codec, verify `bitrate()`.

Existing `test_create_codec`, `test_encode_decode_roundtrip`, `test_wideband_config`, `test_fullband_config`, `test_plc`, `test_name` remain.

## Risks / open items

- `ropus 0.12` is two days old at time of writing. Bit-exactness against the xiph reference is claimed and tested upstream, but we are an early downstream consumer. Mitigation: the unit tests above plus the existing `test_encode_decode_roundtrip` give us a sanity floor; deeper interop confidence comes from the baresip integration tests already in `crates/mdsiprtp/tests/baresip_integration/`.
- **Mid-stream setter audio quality is not verified by us.** We rely on ropus's design claim that runtime mutation of bitrate / FEC / DTX is glitch-free (libopus's equivalent CTLs are documented as safe; ropus tracks that contract). Our tests confirm setters return Ok and getters round-trip — they do not measure audio artifacts across a transition. A PESQ/POLQA-style oracle would be the right tool if a regression ever shows up; out of scope here.
- None of the typed enums introduce lifetime parameters or non-`Copy` payloads, so the API change is mechanical at every call site (`OpusConfig` literals must use `Bitrate::Bits(N)` instead of `N`).
