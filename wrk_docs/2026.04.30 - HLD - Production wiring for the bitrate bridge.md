# HLD — Production wiring for the bitrate bridge

## Why

The bridge HLD (sibling, 2026-04-29) deliberately stopped at `BitrateBridge` and recorded two follow-ups required for the bridge to be more than dead code in production:

1. An inbound RTCP receive path that runs `RtcpCompound::parse` → routes REMB into `CongestionController::on_remb`.
2. Replacing `MediaSession`'s hard-coded `G711Codec` with codec-aware selection so the bridge has an `Opus` encoder to act on.

This HLD does both. After it lands, the only thing standing between an inbound REMB packet and an Opus encoder rate change is the (still-missing) RTP/RTCP socket loop — that's a separate, larger piece of work, explicitly out of scope here.

## Goal

Make `MediaSession` codec-aware; give it a `tick(now)` and a `handle_rtcp(&bytes)` method; have it own the `CongestionController` and `BitrateBridge` for adaptive codecs only. After this HLD, a future socket loop need only:

- Hand inbound RTCP bytes to `media_session.handle_rtcp(&bytes)`.
- Call `media_session.tick(Instant::now())` on roughly a 100 ms cadence.

…and the bridge will close the loop.

## Non-goals

- Building the RTP/RTCP UDP socket loop. No socket binding, no `tokio::interval`, no transport-layer changes. Scope is the API the eventual loop will call.
- Generating outbound REMB. Still consume-only.
- Transport-wide congestion control (RFC 8888 / TCC).
- Wiring `cc.on_nack(...)` and `cc.on_rtt(...)`. RTCP dispatch in v1 handles **REMB only** — it's the headline signal and the only one the bridge currently needs. NACK and RTT are noted as follow-ups.
- Re-INVITE / mid-call codec switch. `MediaSession` is constructed once per `set_negotiated_media`.

## Design

### `SessionCodec` enum (`rsiprtp-session`)

Replace `MediaSession.codec: G711Codec` with a small enum that owns whichever codec was negotiated:

```rust
// crates/rsiprtp-session/src/session_codec.rs
pub enum SessionCodec {
    G711(G711Codec),
    G722(G722Codec),
    Opus(OpusCodec),
}

impl SessionCodec {
    /// Build the concrete codec for an SDP-negotiated entry. Errors on
    /// unsupported encodings — caller decides whether to reject the call.
    pub fn for_negotiated(negotiated: &Codec) -> Result<Self, String>;

    pub fn encode(&mut self, samples: &[i16]) -> Result<Vec<u8>, String>;
    pub fn decode(&mut self, payload: &[u8]) -> Result<Vec<i16>, String>;
    pub fn samples_per_frame(&self) -> usize;

    /// `Some` only for codecs that adapt their encoder rate at runtime.
    pub fn as_adaptive_mut(&mut self) -> Option<&mut dyn AdaptiveBitrate>;
}
```

`for_negotiated` dispatches on `negotiated.encoding` (case-insensitive — real SDP capitalisation varies):

| `encoding` (lower-cased) | Constructed |
|---|---|
| `"pcmu"` | `G711(MuLaw)` |
| `"pcma"` | `G711(ALaw)` |
| `"g722"` | `G722Codec::new()` |
| `"opus"` | `OpusCodec::with_config(OpusConfig::fullband_speech())` (48 kHz / 1 ch / 20 ms / 32 kbps starting rate per RFC 7587 § 7) |
| anything else | `Err(...)` |

`as_adaptive_mut` returns `Some(&mut c)` only on the Opus arm. This is the exact pattern the briefing's option #1 imagined: caller holds an `Option<&mut dyn AdaptiveBitrate>` resolved at codec-selection time. No common `Codec` trait, no no-op methods on G.711 / G.722.

### `MediaSession` surface changes

The struct gains two fields and replaces the old codec one:

```rust
pub struct MediaSession {
    rtp_session: RtpSession,
    jitter_buffer: JitterBuffer,
    codec: SessionCodec,                  // was G711Codec
    adaptive: Option<AdaptiveCongestion>, // None for G.711 / G.722
    remote_addr: Option<SocketAddr>,
    local_port: u16,
    active: bool,
}

struct AdaptiveCongestion {
    cc: CongestionController,
    bridge: BitrateBridge,
}
```

`AdaptiveCongestion` is private and constructed only when the codec adapts. For Opus VoIP the CC is bounded to match the codec's range:

```rust
CongestionController::new(
    32_000,   // initial — matches the encoder's starting rate
    6_000,    // min — Opus standard floor
    128_000,  // max — sensible speech ceiling, retunable per deployment
)
```

G.711 / G.722 sessions get `adaptive = None`. RTCP is still parsed for them (cheap; sets up future SR/RR telemetry) but no adaptation runs.

#### Constructor change

```rust
// Before
pub fn new(ssrc: u32, payload_type: u8, clock_rate: u32, local_port: u16) -> Self;

// After
pub fn for_negotiated(
    ssrc: u32,
    negotiated: &Codec,
    local_port: u16,
) -> Result<Self, String>;
```

Single existing call site is `Call::set_negotiated_media` (`crates/rsiprtp-session/src/call.rs:~405`); it already has `&NegotiatedMedia` in hand. The change ripples one signature up: that function now returns `Result<(), String>` instead of `()`. The caller of `set_negotiated_media` (a SIP event handler in the call layer) likewise becomes fallible — surface the error there rather than swallowing. Two-hop ripple, both straightforward.

#### `encode_audio` becomes fallible

`SessionCodec::encode` returns `Result<Vec<u8>, String>` because G.722 and Opus encoding can fail; G.711 is wrapped as `Ok(...)` for uniformity. `MediaSession::encode_audio` currently returns `RtpPacket` directly — after this HLD it returns `Result<RtpPacket, String>`. Same one-callsite-up ripple as the constructor change. No internal callers to chase: the test in `bitrate_bridge_remb.rs` already calls Opus's fallible `.encode(...)?` directly.

#### Two new public methods

```rust
/// Hand an inbound RTCP compound packet to the session. Parses the bytes
/// and routes feedback (currently REMB only) into the CongestionController
/// when the active codec is adaptive. Errors only on malformed input;
/// unknown / non-actionable RTCP types are silently ignored.
pub fn handle_rtcp(&mut self, bytes: &[u8]) -> Result<(), String>;

/// Periodic tick. Caller invokes ~every 100 ms.
/// Drives `CongestionController::update()` and `BitrateBridge::poll()`.
/// A codec rejection from the bridge is logged at warn and swallowed —
/// it must not fail the call.
pub fn tick(&mut self, now: Instant);
```

### RTCP compound parsing (`rsiprtp-rtp`)

`rsiprtp-rtp::rtcp` already has `RtcpCompound::build(&self) -> Bytes`; today the symmetric parse is missing. Add it:

```rust
impl RtcpCompound {
    /// Parse a compound RTCP packet (RFC 3550 § 6.1). Walks the byte
    /// buffer by header length, calling existing per-packet parse functions
    /// for known types. Unknown types are preserved as `RtcpPacket::Unknown`
    /// (receiver leniency — RFC 3550 § 6.1 last paragraph).
    pub fn parse(data: &[u8]) -> Result<Self, String>;
}
```

Implementation: a 30-50 line loop over the input. Every iteration reads `RtcpHeader::parse`, advances by the declared length, and dispatches to the existing `SenderReport::parse`, `ReceiverReport::parse`, `Sdes::parse`, `Bye::parse`, `Remb::parse` (and whatever else `rsiprtp-rtp` already exposes). Build/parse must round-trip — covered by tests.

### RTCP dispatch (v1 — REMB only)

`MediaSession::handle_rtcp` walks the parsed compound and acts on REMB packets:

```rust
pub fn handle_rtcp(&mut self, bytes: &[u8]) -> Result<(), String> {
    let compound = RtcpCompound::parse(bytes)?;
    let Some(adapt) = self.adaptive.as_mut() else {
        return Ok(()); // Fixed-rate codec: no consumer for feedback yet.
    };
    for packet in compound.packets() {
        if let Some(remb) = packet.as_remb() {       // exact API matches rsiprtp-rtp's enum
            adapt.cc.on_remb(remb.bitrate);
        }
        // SR / RR / NACK / RTT — see Risks (deferred).
    }
    Ok(())
}
```

Variant names depend on `rsiprtp-rtp::rtcp`'s actual enum shape — the implementer matches the real types, not invented ones. The intent is: pull REMB out, call `cc.on_remb`, ignore everything else cleanly.

### Periodic tick

```rust
pub fn tick(&mut self, now: Instant) {
    let Some(adapt) = self.adaptive.as_mut() else {
        return;
    };
    adapt.cc.update();
    if let Some(adaptive_codec) = self.codec.as_adaptive_mut() {
        if let Err(e) = adapt.bridge.poll(adapt.cc.target_bitrate(), adaptive_codec, now) {
            tracing::warn!(error = %e, "BitrateBridge::poll rejected by codec");
        }
    }
}
```

Caller drives `tick` from whatever periodic mechanism they own (eventually a tokio interval; today, a unit test or the future socket loop). The 100 ms cadence is a recommendation; the bridge's own 200 ms hysteresis interval rate-limits codec writes regardless of how often `tick` is called.

Codec rejection from `bridge.poll` is logged and swallowed, per the sibling HLD's "callers should log-and-continue" guidance.

## Implementation outline

1. **New** `crates/rsiprtp-session/src/session_codec.rs` — `SessionCodec` enum, `for_negotiated`, `encode`/`decode`/`samples_per_frame`, `as_adaptive_mut`.
2. `crates/rsiprtp-session/src/lib.rs` — `pub mod session_codec;` + re-export `SessionCodec`.
3. `crates/rsiprtp-session/src/call.rs`:
   - `MediaSession.codec: G711Codec` → `SessionCodec`; add `adaptive: Option<AdaptiveCongestion>`; add `AdaptiveCongestion` private struct.
   - Replace `MediaSession::new` with `for_negotiated(ssrc, &Codec, local_port) -> Result<Self, String>`.
   - Add `handle_rtcp` and `tick`.
   - Update `Call::set_negotiated_media` (~line 405) to use the new constructor and surface the `Result`.
4. `crates/rsiprtp-rtp/src/rtcp.rs` — add `RtcpCompound::parse`.
5. `crates/rsiprtp-session/Cargo.toml` — `rsiprtp-media` is already a regular `[dependencies]` entry (`rsiprtp-media.workspace = true`); add `features = ["opus"]` to it (rewrites the line as `rsiprtp-media = { workspace = true, features = ["opus"] }`). Drop the now-redundant `[dev-dependencies]` line that the integration test added — Cargo merges features so the production dep covers both build and test.

No edits to `rsiprtp-core`, `rsiprtp-transport`, `rsiprtp-sdp`, `rsiprtp-dialog`, or `rsiprtp-transaction`.

## Tests

### Unit — `SessionCodec` (`session_codec.rs`)

- `for_negotiated_routes_pcmu` / `_pcma` / `_g722` / `_opus` — each codec name yields the expected variant.
- `for_negotiated_is_case_insensitive` — `"PCMU"`, `"pcmu"`, `"Pcmu"` all route to the same variant; `"OPUS"` and `"opus"` likewise.
- `for_negotiated_rejects_unknown` — `"AMR"` yields `Err`.
- `as_adaptive_mut_only_for_opus` — Opus variant returns `Some`, the others return `None`.

### Unit — `RtcpCompound::parse` (`rsiprtp-rtp/src/rtcp.rs`)

- `roundtrip_sr_sdes` — build SR + SDES, serialise via `build()`, parse, assert structural equality.
- `roundtrip_rr_remb` — build RR + REMB at 64 kbps, parse, assert REMB bitrate preserved.
- `parse_rejects_truncated` — input shorter than declared length yields `Err`.
- `parse_preserves_unknown_types` — synthesise a compound containing an unknown packet-type code, parse, assert the unknown packet appears (not rejected).

### Integration — `crates/rsiprtp-session/tests/rtcp_drives_bridge.rs` (new)

End-to-end through `MediaSession`, the natural extension of `bitrate_bridge_remb.rs`:

1. Build a `Codec { encoding: "opus", payload_type: 111, clock_rate: 48_000, channels: 2, .. }` (RTP `a=rtpmap:111 opus/48000/2`).
2. `MediaSession::for_negotiated(ssrc, &codec, local_port)?` — codec starts at 32 kbps via `OpusConfig::fullband_speech()`.
3. Encode one frame via `media.encode_audio(...)?`; record encoded payload size.
4. Build a compound RTCP containing a REMB at **16 kbps** (well below the codec's 32 kbps starting rate, and below `0.9 × 32 kbps` so `cc.on_remb` actually reduces target); serialise via `build()`.
5. `media.handle_rtcp(&bytes)?`.
6. `media.tick(Instant::now())`.
7. Encode another frame; assert encoded payload **shrunk** — directional `after_bytes < initial_bytes` plus an absolute floor (e.g. `>= 20` bytes), the same shape as `bitrate_bridge_remb.rs`. The byte delta IS the proof that adaptation occurred end-to-end; no need to expose `MediaSession`'s internal state for a separate "is the codec adaptive" assertion.

A second small test covers the fixed-rate path: `for_negotiated` with `encoding = "PCMU"`, `handle_rtcp` with a REMB packet, `tick`, `encode_audio` — assert no error, no panic, and that frame size is unchanged. This guards against regressions where the RTCP path accidentally tries to mutate a non-existent `adaptive`.

### What we do not test

- `CongestionController` state transitions and `BitrateBridge` hysteresis — already exhaustively covered.
- Opus encode/decode correctness — covered by ropus's RFC 6716 vectors.
- The hypothetical socket loop — out of scope.

## Risks / open items

- **NACK + RTT routing are deferred.** `cc.on_nack()` and `cc.on_rtt()` remain uncalled after this HLD. They need their own dispatch from RR / SR (loss fraction; LSR / DLSR for RTT). The right shape is another arm in `handle_rtcp`'s match — a dozen lines per source. Not adding a half-baked first version here.
- **No production driver yet.** `handle_rtcp` and `tick` have no real caller until a socket loop exists. The integration tests exercise them via direct calls; production wiring waits for the loop. This HLD intentionally stops at the API surface so the loop can land independently.
- **`Call::set_negotiated_media` becomes fallible.** The signature change is invasive at the type level but cheap at the source-line level — exactly one call site to update. If that ripples further (into wherever calls it), surface the error there too rather than swallowing.
- **Opus VoIP defaults are conservative.** `fullband_speech()` (48 kHz / 1 ch / 20 ms / 32 kbps) follows RFC 7587. If a deployment wants `wideband_speech()` (16 kHz, lower CPU), that's a future config knob; not adding one in v1.
- **CC bounds are static (32 / 6 / 128 kbps).** Reasonable for VoIP; if anyone wants per-deployment tuning, they'll need a config struct. Don't add one until that need is real — premature abstraction.
- **No CC for fixed-rate codecs** is deliberate. RR / SR telemetry never reaches a consumer for those calls. If RTT / jitter telemetry on G.711 calls becomes desirable, the easy path is a `Stats` collector beside `AdaptiveCongestion` — orthogonal to the bridge.
